# 参与 SP Incubator 流程

这个文档描述了一个典型的Incubator参与者在项目中的一个参与流程。我们推荐您先阅读一下Incubator项目页面，来对整个项目有个了解。在过程中有任何问题/疑问，请不要犹豫，可在Slack (opens new window)上找到我们。Slack的#fil-venus-cn (opens new window)会是您问题得到回答最佳的方式，并且您的问题也可以帮助其他人找到答案。

# 申请

请在这里 (opens new window)提交Incubator申请。然后等待venus@ipfsforce.com的邮件或者是Venus核心团队在Slack上联系您，来确认申请。

# 测试

这里非常推荐在上主网之前，首先使用在Calibration网络上使用免费Venus链服务 (opens new window)来测试 (opens new window)您的集群。您可以从@Venus-Ops (opens new window)处获取相关使用链服务的权限。

切记测试您集群的封装和提交WindowPost的功能。在没有完全掌握Venus运维之前上主网，可能会因为软件配置/使用不当导致您的财产损失。您可以在任何时候，在Slack的#fil-venus-cn (opens new window)频道中寻求技术支持。

在采购硬件之前，我们推荐您在#fil-venus-cn (opens new window)咨询Venus核心团队或者Venus Master，关于最佳的硬件配置的相关问题来达到最佳体验。

# 孵化和奖励

在成功测试后，可以从@Venus-Ops (opens new window)或者Venus Master处获得使用Incubator链服务的相关权限，并开始在主网给Filecoin网络提供存储。请注意，我们推荐使用 (opens new window)venus-cluster来作为Venus的算力服务模块.venus-cluster将逐步替代原来的venus-sealer。根据孵化器奖励规则 ，您可以在网络奖励的基础上获得额外的孵化器专属奖励。

# 退出

如果在任何Incubator的时间节点您想要退出，或者这个时长1年的项目结束的时候，您可以参照这个文档 (opens new window)或者在#fil-venus-cn (opens new window)咨询我们其他退出方案。
